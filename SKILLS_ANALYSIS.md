## 1. gesture-recognition-system（手势识别系统）

### 技能概述
基于MediaPipe Hands的手势识别系统，支持12+种手势定义，可映射到VRM动画并联动情感系统。

### 架构图
```
┌─────────────────────────────────────────────┐
│         GestureRecognitionSystem            │
├─────────────────────────────────────────────┤
│ - MediaPipe Hands (实时手部追踪)             │
│ - 12+ 手势定义 (thumbs_up, peace, wave等)    │
│ - 手势检查算法 (landmark几何分析)             │
│ - 历史记录管理                               │
└──────────────┬──────────────────────────────┘
               │
┌──────────────▼──────────────────────────────┐
│         GestureToVRMMapper                  │
├─────────────────────────────────────────────┤
│ - 手势到VRM动画映射表                        │
│ - BlendShape权重控制                         │
│ - 头部倾斜/身体倾斜动画                      │
└──────────────┬──────────────────────────────┘
               │
┌──────────────▼──────────────────────────────┐
│         GestureEmotionLinker                │
├─────────────────────────────────────────────┤
│ - 手势到情感映射                             │
│ - 情感状态管理                               │
│ - 情感动画触发                               │
└─────────────────────────────────────────────┘
```

### 关键类及其职责
| 类名 | 职责 |
|------|------|
| `GestureRecognitionSystem` | 主手势检测引擎，管理MediaPipe Hands初始化、手势定义和检测算法 |
| `GestureToVRMMapper` | 将检测到的手势映射到VRM模型的BlendShape和动画 |
| `GestureEmotionLinker` | 将手势关联到情感系统，触发情感动画 |

### 数据流描述
1. **输入**: 视频流 → MediaPipe Hands → 21个手部关键点
2. **处理**: 手势检查算法分析关键点几何关系 → 匹配手势定义
3. **输出**: 手势名称 + 情感 + VRM动画参数 → 触发回调函数

### 核心算法
- **手势检测**: 基于关键点y坐标比较判断手指伸展状态
- **挥手检测**: 基于历史数据方差分析手部运动
- **双手手势**: 检测左右手同时出现或手指接近

### 集成模式
```javascript
// 与情感系统集成
const gesture = new GestureRecognitionSystem({
  onGesture: (gesture) => {
    emotionSystem.setEmotion(gesture.emotion);
    vrmMapper.playGesture(gesture.name);
  }
});

// 与VRM系统集成
const mapper = new GestureToVRMMapper(vrmComponent);
mapper.applyGestureAnimation('thumbs_up');
```

### 性能特征
- **内存使用**: 手势历史限制10条，防止内存泄漏
- **处理频率**: 手势冷却时间500ms，避免频繁触发
- **计算开销**: 每帧检查12+手势算法，复杂度O(n)
- **实时性**: MediaPipe Hands本地处理，无网络延迟

### 安全考虑
- **权限**: 需要相机权限，用户明确授权
- **数据隐私**: 所有处理在本地进行，不发送到外部服务器
- **内存管理**: 限制历史记录长度，防止内存过度使用
- **错误处理**: 手势检测异常被捕获，不影响主流程

### 推荐改进
1. **增加手势识别准确率**: 使用机器学习模型替代硬编码规则
2. **支持更多手势**: 添加OK手势、鼓掌等常见手势
3. **手势动画优化**: 添加平滑过渡和动画混合
4. **性能优化**: 使用Web Worker进行手势计算，避免阻塞主线程

## 2. semantic-memory-system（语义记忆系统）

### 技能概述
基于ChromaDB的向量语义记忆系统，支持RAG检索增强生成，多集合记忆管理。

### 架构图
```
┌─────────────────────────────────────────────┐
│         SemanticMemorySystem                │
├─────────────────────────────────────────────┤
│ - ChromaDB客户端                            │
│ - 多集合管理 (conversations, facts等)        │
│ - 向量嵌入生成 (sentence-transformers)       │
│ - 语义相似度搜索                             │
└──────────────┬──────────────────────────────┘
               │
┌──────────────▼──────────────────────────────┐
│         RAGContextBuilder                   │
├─────────────────────────────────────────────┤
│ - 上下文构建算法                             │
│ - 令牌估算和截断                             │
│ - 多集合结果融合                             │
└──────────────┬──────────────────────────────┘
               │
┌──────────────▼──────────────────────────────┐
│         MemoryConsolidator                  │
├─────────────────────────────────────────────┤
│ - 24小时记忆整合                             │
│ - 主题提取和偏好识别                         │
│ - 人格档案构建                               │
└─────────────────────────────────────────────┘
```

### 关键类及其职责
| 类名 | 职责 |
|------|------|
| `SemanticMemorySystem` | 核心记忆存储和检索，管理ChromaDB集合和向量操作 |
| `RAGContextBuilder` | 为LLM提示构建相关上下文，管理令牌限制 |
| `MemoryConsolidator` | 整合碎片化记忆，构建长期人格档案 |

### 数据流描述
1. **存储流程**: 内容 → 向量化 → 存储到对应集合 → 返回记忆ID
2. **检索流程**: 查询 → 向量化 → 相似度计算 → 阈值过滤 → 返回结果
3. **整合流程**: 定时扫描旧记忆 → 主题提取 → 构建人格档案 → 更新personality集合

### 集合结构
| 集合 | 用途 | 元数据字段 |
|------|------|-----------|
| conversations | 对话历史 | userId, role, timestamp |
| facts | 事实知识 | userId, confidence, source |
| preferences | 用户偏好 | userId, preferenceType, value |
| personality | 人格特征 | userId, trait, intensity |
| events | 重要事件 | userId, eventType, description |

### 集成模式
```javascript
// 与LLM集成
const context = await memory.getContext('用户喜欢什么电影？');
const prompt = `基于以下记忆回答问题：\n${context}\n\n问题：用户喜欢什么电影？`;

// 与对话系统集成
await memory.recordConversation('user123', 'user', '我喜欢科幻电影');
await memory.recordConversation('user123', 'assistant', '好的，我会记住您的偏好');
```

### 性能特征
- **存储性能**: ChromaDB本地持久化，启动时间约1-2秒
- **检索延迟**: 向量搜索<100ms，支持实时查询
- **内存使用**: 取决于记忆数量，每条约1KB
- **嵌入计算**: 使用@xenova/transformers，CPU计算，可能成为瓶颈

### 安全考虑
- **数据隔离**: 每个用户的数据通过元数据隔离
- **隐私保护**: 敏感信息应加密存储
- **访问控制**: 当前无认证机制，需集成到认证系统中
- **数据备份**: ChromaDB文件需定期备份

### 推荐改进
1. **添加缓存层**: 对常用查询结果进行缓存，提高检索速度
2. **异步嵌入计算**: 使用Web Worker进行向量计算
3. **数据加密**: 对敏感记忆内容进行加密存储
4. **记忆衰退**: 实现记忆重要性衰减机制，自动清理过期记忆
5. **分布式支持**: 支持ChromaDB服务器模式，实现跨设备同步

| 指标 | 健康阈值 | 警告阈值 | 危险阈值 |

```javascript
  monitor.recordFrameTime(deltaTime);

}
```

### 性能特征
- **开销**: <1% CPU使用率，主要基于Performance API
- **实时性**: 每秒更新一次，FPS每帧计算
- **内存使用**: 历史数据限制60个点，约10KB
- **网络影响**: ping测试每10秒一次，极小网络开销

### 安全考虑
- **权限**: 需要Performance API权限，现代浏览器支持
- **数据泄露**: 性能数据不包含敏感信息
- **DoS防护**: 监控频率限制，防止过度消耗资源
- **隐私**: 不收集用户身份信息

### 推荐改进
1. **GPU监控**: 添加GPU使用率监控（WebGL扩展）
2. **崩溃报告**: 集成崩溃报告和错误追踪

4. **多设备支持**: 支持移动端性能监控和适配
5. **云同步**: 可选的性能数据云同步和分析

### 关键类及其职责
| 类名 | 职责 |
|------|------|
| `WebRTCVoiceStream` | 主WebRTC连接管理，处理信令、ICE、数据通道 |

| `AudioStreamProcessor` | 音频流处理，增益、压缩、降噪等效果 |

### 数据流描述
1. **发送流程**: 麦克风 → AudioStreamProcessor → WebRTC PeerConnection → 服务器
2. **接收流程**: 服务器 → WebRTC PeerConnection → 音频播放
3. **数据通道**: 双向文本/二进制数据传输（转录文本、TTS请求）

### 信令协议
```javascript
// 信令消息类型
{
  type: 'offer' | 'answer' | 'ice-candidate' | 'transcript' | 'tts-request',
  // 其他字段根据类型不同
}
```

### 集成模式
```javascript
// 与语音识别集成
const voice = new WebRTCVoiceStream({
  onTranscript: (text, isFinal) => {
    if (isFinal) {
      llm.sendQuery(text);
    }
  }
});

// 与TTS集成
voice.sendTTS('你好！', 'happy');
```

### 性能特征
- **延迟**: 端到端延迟<200ms（本地网络）
- **带宽**: 128kbps Opus编码，适合语音
- **连接时间**: 3-5秒建立P2P连接
- **CPU使用**: 音频处理约5-10% CPU

### 安全考虑
- **加密**: WebRTC默认使用DTLS-SRTP加密
- **认证**: 信令服务器应使用WSS和认证
- **权限**: 需要麦克风权限，用户明确授权
- **隐私**: 音频数据在传输过程中加密

### 推荐改进
1. **STUN/TURN服务器**: 自建STUN/TURN服务器，提高连接成功率
2. **自适应码率**: 根据网络状况自动调整音频码率
3. **多房间支持**: 支持多人语音聊天室
4. **回声消除**: 增强的回声消除算法
5. **降噪改进**: 使用AI模型进行背景噪声消除

## 6. elevenlabs-lip-sync（ElevenLabs唇形同步）

### 技能概述
ElevenLabs TTS集成，基于音素时间线的VRM唇形同步，支持30+音素映射。

### 架构图
```
┌─────────────────────────────────────────────┐
│         ElevenLabsLipSync                   │
├─────────────────────────────────────────────┤
│ - ElevenLabs TTS API集成                    │
│ - 音素时间线解析                             │
│ - 浏览器Speech API回退                       │
│ - 音频播放管理                               │
└──────────────┬──────────────────────────────┘
               │
┌──────────────▼──────────────────────────────┐
│         LipSyncToVRM                        │
├─────────────────────────────────────────────┤
│ - 音素到VRM BlendShape映射                   │
│ - 30+音素映射表                              │
│ - 口型形状权重计算                           │
│ - 实时唇形动画                               │
└─────────────────────────────────────────────┘
```

### 关键类及其职责
| 类名 | 职责 |
|------|------|
| `ElevenLabsLipSync` | TTS和音素处理，管理ElevenLabs API和音素时间线 |
| `LipSyncToVRM` | VRM BlendShape映射，将音素转换为唇形动画 |

### 音素映射表
| 音素 | VRM1 Blend | VRM0 Blend | 口型 |
|------|------------|------------|------|
| A, AA, AH | A | a | 张开 |
| E, EH, ER | E | e | 宽 |
| I, IH, IY | I | i | 窄 |
| O, OH, OW | O | o | 圆 |
| U, UH, UW | U | u | 圆 |
| M, B, P | M | mouthClose | 闭合 |
| F, V | F | mouthLower | 降低 |

### 数据流描述
1. **TTS流程**: 文本 → ElevenLabs API → 音频 + 音素时间线 → 唇形动画
2. **回退流程**: 文本 → 浏览器Speech API → 字符分析估算音素 → 基本唇形
3. **动画流程**: 音素时间线 → 时间戳匹配 → BlendShape权重 → VRM更新

### 集成模式
```javascript
// 与VRM组件集成
const lipSync = new LipSyncToVRM(vrmComponent);
lipSync.initialize();

// 语音合成
await lipSync.speak('Hello world!', {
  stability: 0.5,
  similarityBoost: 0.75
});

// 停止
lipSync.stop();
```

### 性能特征
- **延迟**: TTS生成200-500ms，唇形同步实时
- **内存使用**: 音素时间线约1-5KB
- **CPU使用**: 唇形计算<2% CPU
- **网络**: TTS API调用，依赖网络状况

### 安全考虑
- **API密钥**: 存储在环境变量，不硬编码
- **数据隐私**: 仅发送文本，不发送用户语音
- **音频处理**: 本地下载后处理，不持续传输
- **回退机制**: API失败时使用本地浏览器TTS

### 推荐改进
1. **本地TTS**: 集成Web Speech API完全本地化
2. **音素准确度**: 使用更准确的音素识别算法
3. **表情同步**: 与面部表情系统集成
4. **多语言支持**: 增强多语言音素映射
5. **情感识别**: 从文本中识别情感并调整唇形强度

## 7. vrm-animation-system（VRM动画系统）

### 技能概述
VRMA动画文件加载和BVH动作捕捉集成，使用@pixiv/three-vrm-animation库。

### 架构图
```
┌─────────────────────────────────────────────┐
│         VRMAnimationSystem                  │
├─────────────────────────────────────────────┤
│ - GLTFLoader + VRMA插件                     │
│ - BVHLoader (three.js)                      │
│ - AnimationMixer管理                        │
│ - 动画库管理                                 │
└──────────────┬──────────────────────────────┘
               │
┌──────────────▼──────────────────────────────┐
│         VRMAnimationEditor                  │
├─────────────────────────────────────────────┤
│ - 关键帧编辑器                               │
│ - 动画剪辑创建                               │
│ - 导入导出功能                               │
└──────────────┬──────────────────────────────┘
               │
┌──────────────▼──────────────────────────────┐
│         BVHRetargeter                       │
├─────────────────────────────────────────────┤
│ - BVH到VRM骨骼映射                          │
│ - 动作重定向算法                             │
│ - 比例适配                                   │
└─────────────────────────────────────────────┘
```

### 关键类及其职责
| 类名 | 职责 |
|------|------|
| `VRMAnimationSystem` | 主动画系统，管理VRMA和BVH加载、播放和混合 |
| `VRMAnimationEditor` | 基本动画编辑，创建和修改动画关键帧 |
| `BVHRetargeter` | BVH动作捕捉重定向，将通用骨骼映射到VRM |

### 支持格式
| 格式 | 扩展名 | 来源 | 特点 |
|------|--------|------|------|
| VRMA | .vrma | @pixiv/three-vrm-animation | VRM专用动画格式 |
| BVH | .bvh | 动作捕捉 | 通用骨骼动画 |
| FBX | .fbx | Mixamo, Blender | 商业动画格式 |

### 数据流描述
1. **VRMA加载**: .vrma文件 → GLTFLoader → VRMA插件 → AnimationClip
2. **BVH加载**: .bvh文件 → BVHLoader → 骨骼重定向 → AnimationClip
3. **播放流程**: AnimationClip → AnimationMixer → VRM模型更新

### 集成模式
```javascript
// 与VRM组件集成
const vrma = new VRMAnimationSystem(vrmComponent);
await vrma.initialize();

// 加载VRMA
await vrma.loadVRMA('/animations/greeting.vrma', 'greeting');

// 加载BVH
await vrma.loadBVH('/mocap/dance.bvh', 'mocap_dance');

// 播放
vrma.playWithCrossFade('dance', 0.3);
```

### 性能特征
- **加载时间**: VRMA文件50-200ms，BVH文件100-500ms
- **内存使用**: 每个动画剪辑约20-100KB
- **CPU使用**: 动画混合<5% CPU
- **平滑度**: 60fps，支持多动画混合

### 安全考虑
- **文件验证**: 加载的动画文件应进行格式验证
- **大小限制**: 限制动画文件大小，防止内存溢出
- **错误处理**: 加载失败时有降级方案
- **沙箱**: 动画编辑器应在沙箱环境中运行

### 推荐改进
1. **动画压缩**: 支持动画数据压缩算法
2. **实时预览**: 在编辑器中实时预览动画效果
3. **动画融合**: 支持更复杂的动画融合算法
4. **物理模拟**: 集成物理引擎模拟布料和头发
5. **云端动画库**: 支持从云端下载动画资源

### 数据流集成
1. **输入处理**: 语音(WebRTC) + 手势(MediaPipe) → 情感分析 → VRM动画

### 性能优化技术
1. **Web Workers**: 将计算密集型任务（手势检测、向量计算）移到Worker线程
2. **动画合并**: 将多个动画剪辑合并为单个剪辑，减少绘制调用
3. **LOD系统**: 根据距离自动切换动画和模型质量
4. **缓存策略**: 缓存频繁使用的动画、音素映射、手势定义
5. **资源释放**: 及时释放不再使用的动画剪辑和音频缓冲区

### 安全措施
1. **权限管理**: 明确请求相机、麦克风权限，提供权限状态回调
2. **数据加密**: WebRTC使用DTLS-SRTP，API密钥使用环境变量
3. **输入验证**: 所有外部数据（动画文件、API响应）进行验证
4. **内存管理**: 限制历史记录、缓存大小，防止内存泄漏
5. **错误处理**: 所有异步操作有错误处理，防止系统崩溃

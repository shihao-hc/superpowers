# MCP 功能操作手册

## 一、工具市场

### 功能说明

工具市场列出了所有可用的 MCP 工具（如文件操作、GitHub 管理、浏览器调试等），每个工具都标注了使用提示，帮助你安全高效地完成操作。

### 使用方法

1. 进入 MCP 控制台 → 点击 **工具市场**

2. 工具列表会显示工具名称、描述以及三个关键提示：
   - **只读**：该工具不会修改任何数据，可放心使用
   - **破坏性**：该工具可能删除或覆盖数据，执行前会要求二次确认
   - **幂等**：重复执行相同操作不会产生副作用

3. 点击工具右侧的 **执行** 按钮，填写参数后即可运行

4. 对于破坏性工具，系统会弹出确认框，确认后才会真正执行

## 二、Dry-run 预览

### 功能说明

在执行写操作（如编辑文件、创建 Issue）之前，可以先预览操作的影响，确认无误后再执行，避免误操作。

### 使用方法

1. 在工具市场中，选择支持 dry-run 的工具（如 `edit_file`、`create_issue`）

2. 填写参数后，先点击 **预览** 按钮（而非直接执行）

3. 系统会返回一个预览界面：
   - 对于文件编辑，显示代码差异对比（diff 视图）
   - 对于 Issue 创建，展示标题、内容、标签等信息

4. 若预览结果符合预期，点击 **确认执行** 按钮，操作才会真正生效

5. 所有预览历史可在 **Dry-run 历史** 页面查看，方便追溯

## 三、思维链图谱

### 功能说明

思维链图谱帮助你记录和分析复杂问题的推理过程，支持分支、反思和回溯，适合用于需求分析、问题排查等场景。

### 使用方法

1. 进入 **思维链图谱** 页面

2. **创建新链**：输入初始想法或问题，点击"创建"即可生成一个根节点

3. **添加步骤**：选中一个节点，点击"添加步骤"，输入推理内容，可生成子节点

4. **创建分支**：当有多个可能性时，选中节点后点击"创建分支"，可以并行探索不同路径

5. **添加反思**：在某个步骤上点击"反思"，可以记录对该步骤的质疑、修正或补充说明

6. **回溯执行**：如果某个分支的推理被证明错误，可选中该分支的起点，点击"回溯"，系统会从该点重新开始

7. **保存为笔记**：点击"保存为 Memo"，可将当前思维链内容一键保存到笔记系统中，便于知识沉淀

## 四、Roots 管理（文件沙箱）

### 功能说明

Roots 管理用于控制哪些本地目录可以被工具访问，以及访问权限（只读/可写），保护你的文件安全。

### 使用方法

1. 进入 **Roots 管理** 页面

2. 列表显示当前已授权的根目录及权限

3. **添加根目录**：点击"添加"，选择本地文件夹，并勾选权限（读/写），然后确认

4. **删除根目录**：点击目录旁的删除按钮，确认后该目录将不再被任何工具访问

5. **临时沙箱**：如果需要临时工作区，点击"创建沙箱"，系统会生成一个临时目录，操作完成后可手动清理或自动删除

## 五、安全机制

### 路径限制

- 工具只能访问 Roots 内配置的目录
- 超出范围的路径访问会被拒绝

### 操作预览

- 所有写操作都支持 dry-run 预览
- 预览结果符合预期后再执行

### 破坏性操作确认

- 删除文件、删除目录等操作需要二次确认
- 执行前会明确提示操作后果

---

## 六、故障排除

### 常见问题

| 问题 | 解决方法 |
|------|----------|
| 工具执行失败 | 检查路径是否在 Roots 内 |
| 页面加载缓慢 | 刷新页面或检查网络连接 |
| 无法预览操作 | 检查工具参数格式是否正确 |
| MCP 服务器离线 | 重启服务器或检查配置 |

### 获取帮助

- 查看服务器日志
- 检查 MCP 状态：`/api/mcp/status`
- 检查健康状态：`/api/mcp/health`

Edge Deployment Guide (Phase 10)
=================================
- 目标：在边缘节点和多区域部署 AI 平台，确保低延迟、容错和灾备能力。
- 核心要点：
  - 服务分层：前端静态资源和缓存落在边缘节点，后端 API 在就近区域与中心化区域组合部署。
  - 推理端点就近：区域就近的推理入口，通过 INFER_ENDPOINT 动态路由到就近的推理服务以降低延迟。
  - 数据一致性与缓存：跨区域缓存失效策略、最近记忆同步策略以及分布式缓存一致性设计。
  - 部署方案：Kubernetes 为多区域部署的主线，结合环境代理实现智能路由。
- 验证与运维：跨区域健康检查、全局告警、灾备路径、滚动更新与可观测性。
- 参考资源：整体设计文档、Part-10 边缘部署草案、Part-6 DAG、Part-5 插件等衔接点。
